# Part E: Security Recommendations

Based on the risk assessment and threat analysis above, the following 10 recommendations would meaningfully improve the college's cybersecurity posture.

## 1. Enable Multi-Factor Authentication (MFA)
Even if a password is stolen, MFA blocks unauthorized access by requiring a second verification step — one of the highest-impact, lowest-cost security improvements available.

## 2. Regular System Updates & Patch Management
Many major breaches (including WannaCry) exploited vulnerabilities that already had patches available. Keeping systems updated closes known security gaps before attackers can exploit them.

## 3. Strong Password Policies
Enforcing minimum length, complexity, and regular password changes reduces the risk of accounts being compromised through guessing or brute-force attacks.

## 4. Employee & Student Security Awareness Training
Most breaches start with human error — clicking a phishing link, reusing a password. Regular training builds a "human firewall" that complements technical controls.

## 5. Regular, Tested Backups
Maintaining offline or isolated backups ensures that even in a ransomware attack, the college can restore data without paying a ransom or losing critical records permanently.

## 6. Network Segmentation
Dividing the network into isolated zones (e.g., student records separate from general Wi-Fi) limits how far an attacker or malware can spread if one segment is compromised.

## 7. Role-Based Access Control (Least Privilege)
Restricting access so users only see what they need for their role minimizes the damage that can be done by a compromised or malicious account.

## 8. Email Filtering & Anti-Phishing Tools
Automated filtering reduces the number of phishing emails that ever reach a user's inbox, lowering the chance of human error leading to a breach.

## 9. Incident Response Plan
Having a clear, practiced plan for what to do during a breach (who to contact, how to contain it, how to communicate) significantly reduces response time and damage when an incident occurs.

## 10. Regular Security Audits & Vulnerability Scanning
Periodically testing systems for weaknesses — before attackers find them — helps the college stay ahead of emerging threats rather than reacting after a breach has already happened.
